# 2026-03-10 记忆日志

## 业务规则

### 安全操作（需密码验证）
- 密码存储在系统中，不在对话中显示
- 以下危险操作必须先确认密码：
  1. 清空所有商品/数据
  2. 批量删除商品
  3. 删除采购/销售记录
  4. 批量修改价格或库存
  5. 其他不可逆的数据操作
- 执行前要求用户输入密码，验证通过后执行
- **绝不在对话中显示密码明文**

### 数据库结构变更流程
- **所有创建表或更新表字段的操作必须遵循以下流程：**
  1. 先列出变更计划（新建表/新增字段/修改字段/删除字段等）
  2. 说明变更原因和影响
  3. 等待用户确认同意
  4. 用户同意后才可执行 DDL 操作
- 未经确认不得执行任何 CREATE TABLE / ALTER TABLE / DROP TABLE 语句

### 数据兼容性原则
- **有数据的表：只能新增字段，不能删除或修改字段**
- **空表：可以自由变更结构**
- 变更前必须检查表中是否有数据
- 如果需要调整有数据表的结构，必须先提出数据迁移方案，经用户确认后执行
- 新增字段时需提供默认值或允许 NULL，确保现有数据兼容
